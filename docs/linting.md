### Обраний лінтер та причини вибору

У межах даної лабораторної роботи було обрано <code>Ruff</code>.
<code>Ruff</code> інструмент статичного аналізу, що поєднує функції лінтера та форматувальника,
швидко працює на великих Python-кодових базах і підтримує конфігурацію через <code>pyproject.toml</code>.
Використання цього інструмента є доцільним для проєкту,
оскільки він дозволяє одночасно контролювати дотримання стилістичних правил,
виявляти потенційні помилки, спрощувати структуру коду
та підтримувати його єдине оформлення в межах усієї системи.

### Базові правила Ruff та їх пояснення

#### Категорії активованих правил

- E — pycodestyle errors. Перевірка відповідності коду базовим правилам стилю Python (PEP 8).
- F — Pyflakes. Пошук логічних і синтаксичних проблем без виконання коду.
- I — isort. Автоматична перевірка та сортування імпортів.
- UP — pyupgrade. Перевірка використання сучасного синтаксису Python.
- B — flake8-bugbear. Пошук потенційно небезпечних або підозрілих патернів.
- SIM — flake8-simplify. Пошук надмірно складних конструкцій, які можна спростити.

### Інструкцію з запуску лінтера

Для серверної частини:

- Активувати віртуальне середовище Python:

        .\server\.venv\Scripts\activate

- Виконати перевірку:

       ruff check server

Для телеграм боту:

- Активувати віртуальне середовище Python:

        .\telegram-bot\.venv\Scripts\activate

- Виконати перевірку:

       ruff check telegram-bot
